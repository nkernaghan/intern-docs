Week 1 Curriculum
AI 101 — Five Days
1
Monday
What is an LLM?
A large language model is a neural network trained on massive amounts of text. It predicts the next token in a sequence — which is how it generates, summarizes, and analyzes language. Key concepts: tokens, context windows, temperature, and hallucination. By end of day you should be able to explain to a non-technical person how ChatGPT or Claude works.
2
Tuesday
Embeddings & Vector Search
Embeddings convert text into numerical vectors that capture semantic meaning. Similar concepts end up close together in vector space. This is the foundation of search, recommendation, and RAG systems. Key concepts: cosine similarity, vector databases, Pinecone, Supabase pgvector.
3
Wednesday
Retrieval-Augmented Generation (RAG)
RAG combines a retrieval system with an LLM. Instead of relying on the model's training data, you first retrieve relevant documents and inject them into the prompt. This is how you build AI tools that work on your own internal data — like Haven256 case files, contracts, and intelligence reports.
4
Thursday
APIs & Data Pipelines
An API is how software systems communicate. At Haven256 you will pull data from blockchain APIs (Helius, Alchemy, Blockchair), pass it through a processing pipeline, store it in a database, and surface it through a dashboard or AI tool. Understanding how data flows end-to-end is non-negotiable.
5
Friday
AI Agents
An AI agent goes beyond generating text — it can take actions, call tools, browse data, write code, and complete multi-step tasks autonomously. Agents use LLMs as a reasoning engine and connect them to external tools and APIs. This is where the field is heading and what most of your projects will use.
03
Hands-On
Week 1 Labs
Lab 01
Build a Simple RAG System
Take a set of Haven256 documents, embed them using Claude or OpenAI embeddings, store them in a vector database, and build a simple Q&A interface that retrieves relevant chunks before answering. You should be able to ask it a question about the company and get a grounded answer back.
GOAL — Understand how RAG works end-to-end. Know what happens at each step.
Lab 02
Call a Blockchain API
Use the Helius or Alchemy API to pull the transaction history of a sample wallet address. Parse the JSON response, extract key fields (sender, receiver, amount, timestamp), and store the results in a simple database table. Print a clean summary to the terminal.
GOAL — Get comfortable pulling and parsing real blockchain data.
Lab 03
Prompt Engineering Challenge
Given a raw blockchain transaction record, write a series of prompts to get Claude to: (1) summarize the transaction in plain English, (2) flag anything suspicious, and (3) suggest next investigative steps. Iterate at least 5 times. Document what changed and why each iteration improved.
GOAL — Understand that prompt quality drives output quality. Learn through iteration.
